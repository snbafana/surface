# `clipboardrules` / Copy History Rules Spec

## Why This Matters

Clipboard history is useful because it remembers what the user copied, but it is risky for exactly the same reason. Passwords, transient pasteboard values, giant copied blobs, app-generated snippets, remote clipboard values, and one-off sensitive copies should not be silently persisted forever.

Raycast, Alfred, Maccy, and Keyboard Maestro all validate adjacent clipboard workflows: retention settings, disabled applications, ignored pasteboard types, pause/ignore-next controls, favorite/pin behavior, and clipboard-filter actions. The Surface version should take the privacy and noise-reduction parts, but not become a second clipboard manager, script trigger system, or routing engine.

## Existing Owner / Dedup Decision

- Copy History already owns passive pasteboard polling through `NSPasteboard.general.changeCount`.
- Copy History already owns `copyhistory.txt`, normalized text storage, copyback, clear, and the live polling task.
- `linkinbox` owns URL triage and URL metadata.
- `snippetprompt` owns durable reusable snippets/templates.
- `texttransform` owns deterministic text transforms over explicit input.
- `scriptoutput` owns programmable commands and shell execution.
- `keyboardmaestrobridge` and `hammerspoonbridge` own external automation trigger handoffs.
- `permissionsdashboard` owns permission/state explanations.
- `clipboardrules` should be implemented as a Copy History-owned rules layer, not as a new `BlockRuntime`, package target, registry entry, daemon, or second pasteboard watcher.

Target the existing `plugins/copyhistory/source/Plugin.swift` owner unless implementation proves a tiny helper type should be extracted inside the same target.

## Product Boundary

It should:

- Read `copyhistory-rules.json` next to `copyhistory.txt`.
- Evaluate a small capture decision before writing a new history entry.
- Ignore transient/concealed/generated pasteboard marker types by default.
- Ignore explicitly configured pasteboard types and source bundle IDs when the source marker is present.
- Enforce a maximum text size before persistence.
- Support `pause capture` and `ignore next copy` controls inside Copy History.
- Show rule status and metadata-only blocked counts.
- Keep deterministic preview fixtures through `Block.Context.storageDirectory`.

It should not:

- Add a separate `clipboardrules` plugin target.
- Poll `NSPasteboard.general` outside the existing Copy History runtime.
- Store ignored clipboard text in another log.
- Read selected text, browser tabs, active app contents, or screen/OCR content.
- Infer source app by Accessibility, Input Monitoring, event taps, AppleScript, or shell commands.
- Route clipboard entries into other plugins automatically.
- Execute scripts, Shortcuts, Keyboard Maestro macros, Hammerspoon Lua, or webhooks.
- Provide a programmable condition/action rule engine.
- Transform captured text before storing it.
- Fetch network metadata.
- Auto-save clipboard entries as snippets, bookmarks, tasks, files, or notifications.

## First Version

### Data Modes

Fixture mode:

1. Read `Block.Context.storageDirectory/copyhistory.txt`.
2. Read optional `Block.Context.storageDirectory/copyhistory-rules.json`.
3. Read optional `Block.Context.storageDirectory/copyhistory-rule-events.jsonl` for metadata-only preview counters.
4. Do not read or write the real pasteboard.

Live mode:

1. Continue using the existing Copy History Application Support location.
2. Read/write `copyhistory-rules.json` under the same `Surface/CopyHistory` directory.
3. Evaluate rules inside the existing `captureClipboardIfChanged` flow.
4. Persist only accepted text entries to `copyhistory.txt`.
5. Persist optional blocked-event metadata only when writes are allowed; never persist blocked clipboard text.

### Rules File

```json
{
  "version": 1,
  "updatedAt": "2026-06-22T02:58:53Z",
  "capturePaused": false,
  "ignoreNextCopy": false,
  "maxCharacters": 12000,
  "ignoredPasteboardTypes": [
    "org.nspasteboard.TransientType",
    "org.nspasteboard.ConcealedType",
    "org.nspasteboard.AutoGeneratedType",
    "com.agilebits.onepassword",
    "com.apple.is-remote-clipboard"
  ],
  "ignoredSourceBundleIDs": [
    "com.apple.Passwords",
    "com.agilebits.onepassword7"
  ],
  "maskSuspiciousPreviews": true,
  "blockedEventRetentionDays": 7
}
```

### Blocked Event File

Use metadata only:

```jsonl
{"at":"2026-06-22T02:58:53Z","reason":"concealedPasteboardType","sourceBundleID":"com.apple.Passwords","textLength":18}
{"at":"2026-06-22T02:59:10Z","reason":"oversizeText","textLength":91234}
```

Do not write a preview, hash, excerpt, or transformed copy of blocked text.

### Capture Decision Model

```swift
struct CopyHistoryRules: Codable, Equatable {
    var version: Int
    var updatedAt: Date
    var capturePaused: Bool
    var ignoreNextCopy: Bool
    var maxCharacters: Int
    var ignoredPasteboardTypes: [String]
    var ignoredSourceBundleIDs: [String]
    var maskSuspiciousPreviews: Bool
    var blockedEventRetentionDays: Int
}

struct CopyHistoryCaptureCandidate: Equatable {
    var text: String
    var pasteboardTypes: Set<String>
    var sourceBundleID: String?
    var copiedAt: Date
}

struct CopyHistoryCaptureDecision: Equatable {
    var shouldStore: Bool
    var reason: CopyHistoryCaptureReason
    var displayStatus: String
}

enum CopyHistoryCaptureReason: String, Codable {
    case accepted
    case paused
    case ignoreNext
    case emptyText
    case duplicateHead
    case concealedPasteboardType
    case transientPasteboardType
    case ignoredPasteboardType
    case ignoredSourceBundle
    case oversizeText
}
```

Keep the decision pure and testable. `CopyHistoryStore.add` can keep dedupe/limit responsibility, while a new small rules evaluator decides whether `add` should be called.

## Implementation Notes

Use existing AppKit pasteboard APIs:

- `NSPasteboard.changeCount` remains the existing polling signal.
- `NSPasteboard.string(forType: .string)` remains the text read path.
- `NSPasteboard.types` and `NSPasteboardItem.types` can expose marker/type identifiers for rules.
- `org.nspasteboard.TransientType`, `org.nspasteboard.ConcealedType`, `org.nspasteboard.AutoGeneratedType`, and `org.nspasteboard.source` should be respected when present.
- Existing proprietary markers such as `com.agilebits.onepassword`, `com.typeit4me.clipping`, `de.petermaurer.TransientPasteboardType`, and `Pasteboard generator type` should be defaults.

Do not add active-app detection in v1. If source app exclusion beyond `org.nspasteboard.source` becomes necessary, decide later whether it belongs in Copy History, `contextcard`, or a small shared frontmost-app reader after implementation proves the overlap.

## Display Rules

Header:

- `Watching`, `Paused`, or `Rules active`
- entry count
- ignored count from metadata-only blocked events
- warning if rule file is invalid

Controls:

- pause/resume capture
- ignore next copy
- clear history
- copy rules file path
- reveal rules file

Rows:

- keep existing copied-text rows and copy icon behavior
- optionally mask suspicious-looking previews when `maskSuspiciousPreviews` is enabled
- do not display blocked text

Rules summary:

- max character limit
- enabled default ignored types count
- configured source bundle ignore count
- last blocked reason and time, metadata only

## Actions

- Pause/resume capture.
- Ignore next copy.
- Clear history using the existing Copy History clear path.
- Copy/reveal rules file path.
- Reset rules to defaults if writes are enabled.

No action should paste into the active app, transform text, route records to other plugins, run scripts/macros, change system clipboard privacy settings, or request permissions.

## Runtime Shape

Target: existing `plugins/copyhistory/source/Plugin.swift`

Runtime changes:

1. `start()`: load history and rules; keep the existing polling loop.
2. `refresh()`: reload history/rules and capture only through the existing path.
3. `captureClipboardIfChanged()`: build a capture candidate, evaluate rules, and call `CopyHistoryStore.add` only when accepted.
4. `copy(_:)`: keep existing explicit pasteboard write behavior and update `observedChangeCount`.
5. `makeView()`: add compact rules status and pause/ignore-next controls.

Do not add `plugins/clipboardrules`, `ClipboardRules.Plugin.block`, a new package target, or a `Blocks.registry` entry.

## Fixture Plan

Extend Copy History preview fixtures rather than creating a new plugin fixture set:

- `empty`: existing empty history.
- `mixed-clipboard`: existing normal history.
- `rules-paused`: capture paused, history visible.
- `rules-filtered`: normal entries plus blocked metadata counts.
- `rules-invalid`: invalid rule file state.
- `rules-sensitive`: masked previews and ignored-type summary.

If the preview harness needs fixture names to stay plugin-local, these remain `copyhistory` fixtures.

## Test Plan

- Default rules ignore `org.nspasteboard.TransientType`.
- Default rules ignore `org.nspasteboard.ConcealedType`.
- Default rules ignore `org.nspasteboard.AutoGeneratedType`.
- Proprietary password/temporary markers are ignored by default.
- `org.nspasteboard.source` can exclude a configured bundle ID without active-app inspection.
- Oversize text is not persisted.
- Paused capture does not persist text.
- Ignore-next consumes exactly one changed pasteboard event, then resets.
- Ignored events never store text, hashes, excerpts, or transformed content.
- Accepted text still uses existing Copy History trim/dedupe/limit behavior.
- Fixture mode never reads or writes the real pasteboard.
- No code path runs scripts, macros, network fetches, Accessibility, Input Monitoring, AppleScript, or shell commands.
- Existing `copyhistory` preview fixtures still render nonblank PNGs; new rule fixtures render nonblank PNGs.

## Recommendation

Implement `clipboardrules` as Copy History Rules inside the existing Copy History block. The useful v1 is capture hygiene: default ignored pasteboard markers, pause/ignore-next, max length, source-marker exclusion, and metadata-only blocked counters. Anything that transforms, routes, scripts, enriches, or indexes clipboard content belongs to existing Surface owners or a later explicit spec.
